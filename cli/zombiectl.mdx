## Overview

| Group | Commands |
|-------|----------|
| Authentication | `login`, `logout` |
| Zombies | `install`, `up`, `list`, `status`, `kill`, `logs` |
| Credentials | `credential add`, `credential list` |
| Workspaces | `workspace add`, `workspace list`, `workspace use`, `workspace show`, `workspace credentials`, `workspace remove`, `workspace upgrade-scale`, `workspace billing` |
| External agents | `agent create`, `agent list`, `agent delete` |
| Integration grants | `grant list`, `grant revoke` |
| Diagnostics | `doctor` |

Every command accepts the [global flags](/cli/flags). Every command that returns structured data supports `--json` for machine-readable output.

## Authentication

### `zombiectl login`

Authenticate via Clerk OIDC device flow. Opens your default browser to complete sign-in.

```bash
zombiectl login
```

Use `--no-open` to print the auth URL instead of opening a browser automatically. Use `--timeout-sec` and `--poll-ms` to tune the session poll.

### `zombiectl logout`

Clear the local session token.

```bash
zombiectl logout
```

## Zombies

### `zombiectl install --from <path>`

Scaffold a zombie from a sample directory. Writes `SKILL.md` and `TRIGGER.md` into the active workspace. The `--from` flag is required — there are no bare-name template aliases.

```bash
zombiectl install --from ~/.config/usezombie/samples/platform-ops
```

The `samples/` directory ships with `zombiectl` and contains the flagship `platform-ops` sample. The host-agent install skill (`/usezombie-install-platform-ops`) drives this command under the hood. See [Skills catalog](/skills) and [Templates](/zombies/templates).

### `zombiectl list`

List zombies in the active workspace. Keyset-paginated over `(created_at, id)` — the dashboard `/zombies` page uses the same contract.

```bash
zombiectl list --limit 20
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--workspace-id <id>` | *(active)* | Override the active workspace for this call. |
| `--limit <n>` | `20` | Page size. Clamped to `100`. |
| `--cursor <cursor>` | *(none)* | Cursor from a previous response's `cursor` field. |
| `--json` | *off* | Emit the raw `{items, total, cursor}` envelope for scripting. |

Human-readable output prints a table and, when more pages remain, a "Next" hint with the exact cursor to pass back.

### `zombiectl status`

List every zombie in the current workspace with state, events processed, and budget consumed.

```bash
zombiectl status
```

### `zombiectl kill <zombie_id>`

Stop a zombie by its ID. State is checkpointed; nothing on the event stream is lost.

```bash
zombiectl kill zmb_2041
```

### `zombiectl events <zombie_id>`

Stream the activity stream for one zombie. Replaces the older `zombiectl logs --zombie` form (still accepted as an alias).

```bash
zombiectl events zmb_2041 --follow                      # tail SSE in real time
zombiectl events zmb_2041 --since 2h --limit 100
zombiectl events zmb_2041 --actor 'webhook:*' --json    # filter + machine-readable
```

| Flag | Default | Purpose |
|------|---------|---------|
| `--follow` | off | Tail the SSE stream in real time. |
| `--actor <glob>` | *(none)* | Filter by actor — `webhook:*`, `cron:*`, `steer:*`, `continuation:*`. |
| `--since <duration>` | `24h` | Time window — e.g. `2h`, `7d`. |
| `--cursor <cursor>` | *(none)* | Pagination cursor from a previous response. |
| `--limit <n>` | `50` | Max events per page (clamped to `200`). |
| `--json` | off | Emit raw JSON for scripting. |

### `zombiectl steer <zombie_id> "<message>"`

Post a steer message — a manual trigger that lands on the event stream as `actor=steer:<user>`. Same reasoning loop as a webhook event; useful for ad-hoc investigation or before a webhook is wired up.

```bash
zombiectl steer zmb_2041 "morning health check — anything red?"
```

The reply streams back live; the result also persists on `core.zombie_events` for replay.

## Credentials

### `zombiectl credential add <name> --data='<json>'`

Store a JSON-shaped secret in the current workspace vault. The agent references it by name from `TRIGGER.md`.

Pipe JSON on stdin — never via `--data='<json>'` directly, because that puts the secret in shell history and process argv:

```bash
zombiectl credential add github --data @- <<'JSON'
{ "api_token": "ghp_...", "webhook_secret": "<32-byte hex>" }
JSON
```

Default behaviour is skip-if-exists. Use `--force` to overwrite for rotation.

See [Workspace credentials](/zombies/credentials) for the full vault model.

### `zombiectl credential show <name>`

Print metadata (created, last-rotated). **Values are never returned.**

### `zombiectl credential delete <name>`

Remove a credential from the workspace vault. Any zombie still referencing it will fail with `ERR_ZOMBIE_CREDENTIAL_MISSING` on the next event until you re-add (or update `TRIGGER.md`).

### `zombiectl credential list`

List credential names and creation timestamps in the current workspace. Values are never returned.

```bash
zombiectl credential list
```

## Workspaces

### `zombiectl workspace add <repo-url>`

Connect a GitHub repository as a workspace.

```bash
zombiectl workspace add https://github.com/org/repo
```

The CLI opens the GitHub App install page; after install, the workspace is bound automatically. Use `--default-branch <branch>` to override the default branch (defaults to `main`).

### `zombiectl workspace list`

List all configured workspaces with an active marker.

```bash
zombiectl workspace list
```

### `zombiectl workspace use <id>`

Set the active workspace persistently. Writes `current_workspace_id` to `~/.config/zombiectl/workspaces.json`; every subsequent command that accepts `--workspace-id` defaults to this value when the flag is omitted.

```bash
zombiectl workspace use ws_abc123
```

Errors out if the id isn't in the local list (run `zombiectl workspace list` first). Mission Control's own `active_workspace_id` cookie and the CLI config are independent — switching one does not affect the other. See [Workspaces overview](/workspaces/overview).

### `zombiectl workspace show`

Show the active workspace (or a specific one via `--workspace-id`) — mirrors the Mission Control **Settings** page.

```bash
zombiectl workspace show
```

Prints the workspace id, active marker, repo URL, and default branch. Use `--json` for scripting.

### `zombiectl workspace remove <id>`

Remove a workspace from local config.

```bash
zombiectl workspace remove ws_abc123
```

### `zombiectl workspace upgrade-scale`

Upgrade a workspace to the Scale plan.

```bash
zombiectl workspace upgrade-scale --workspace-id ws_abc123 --subscription-id sub_xxx
```

### `zombiectl workspace billing`

Show the billing summary for a workspace over a given period (default 30 days).

```bash
zombiectl workspace billing --workspace-id ws_abc123 --period 30d
```

## External agents

External agents are API-key–authenticated callers (LangGraph, CrewAI, Composio, your own code) that drive a specific zombie programmatically.

### `zombiectl agent create`

Mint an API key bound to one zombie.

```bash
zombiectl agent create --workspace ws_abc --zombie zm_xyz --name my-agent
```

The raw key (`zmb_…`) is returned **once**; store it in your secret manager.

### `zombiectl agent list`

List every external agent key in a workspace.

```bash
zombiectl agent list --workspace ws_abc
```

### `zombiectl agent delete <agent-id>`

Revoke an external agent key.

```bash
zombiectl agent delete --workspace ws_abc ag_01JABC...
```

## Integration grants

Integration grants are per-zombie permissions to reach a third-party service (Slack workspace, GitHub repo) that the zombie requested interactively.

### `zombiectl grant list --zombie <id>`

List pending and approved grants for a zombie.

```bash
zombiectl grant list --zombie zm_01JABC...
```

### `zombiectl grant revoke --zombie <id> <grant-id>`

Revoke a granted integration. Further attempts by the zombie to reach that integration fail with `UZ-GRANT-003`.

```bash
zombiectl grant revoke --zombie zm_01JABC... gr_01JXYZ...
```

## Diagnostics

### `zombiectl doctor`

Verify API connectivity, authentication, and local configuration. Does not require authentication.

```bash
zombiectl doctor
```

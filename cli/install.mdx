## Install

<Steps>
  <Step title="Install Node.js">
    If you don't have Node.js installed yet, [download it here](https://nodejs.org).
  </Step>

  <Step title="Install with npm">
    Run the following command in your terminal:

    ```bash
    npm install -g @usezombie/zombiectl
    ```

    Use `npx` to run without a global install:

    ```bash
    npx @usezombie/zombiectl --help
    ```
  </Step>
</Steps>

## Agent Skills

The `usezombie-install-platform-ops` skill is the guided install UX driven from a host agent (Claude Code, Amp, Codex CLI, OpenCode). It is served as a single host-neutral SKILL.md from `https://usezombie.sh/skills.md` — no npm package, no per-host plugin manifest.

Drop it into your host's skills folder:

```bash
# Claude Code — adjust path for Amp / Codex CLI / OpenCode
mkdir -p ~/.claude/skills/usezombie-install-platform-ops
curl -fsSL https://usezombie.sh/skills.md \
  -o ~/.claude/skills/usezombie-install-platform-ops/SKILL.md
```

The skill drives `zombiectl install --from`, `zombiectl credential`, `zombiectl steer`, `zombiectl logs`, and the doctor preflight under the hood. See [Quickstart](/quickstart) for the end-to-end flow.

## Verify installation

```bash
zombiectl --version
```

## Check connectivity

Run the built-in diagnostics command to verify API connectivity, auth status, and local configuration:

```bash
zombiectl doctor
```

`doctor` runs three checks and exits non-zero if any fails:

| Check | Verifies |
|---|---|
| `server_reachable` | `GET /healthz` against the configured API URL returns `{ "status": "ok" }` within 5s. |
| `workspace_selected` | The local config has a `current_workspace_id` (set by `zombiectl workspace add` or `workspace use`). |
| `workspace_binding_valid` | The auth token can read the selected workspace's zombies — i.e. it is bound to the right tenant. |

Pass `--json` for the structured `{ ok, api_url, checks: [...] }` envelope. Run it before any install — host-agent skills depend on this preflight.

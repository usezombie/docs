## Response format

Every `4xx` and `5xx` response uses `Content-Type: application/problem+json`:

```json
{
  "docs_uri": "https://docs.usezombie.com/api-reference/error-codes#UZ-ZMB-009",
  "title": "Zombie not found",
  "detail": "No zombie with id 'abc123' in this workspace.",
  "error_code": "UZ-ZMB-009",
  "request_id": "req_a1b2c3d4e5f6"
}
```

| Field | Description |
|---|---|
| `docs_uri` | Stable link to this page for the specific code |
| `title` | Short label — identical for every occurrence of a given code |
| `detail` | Instance-specific context (varies per call) |
| `error_code` | Machine-readable code. Use this for programmatic handling. |
| `request_id` | Correlation ID for support and log tracing |

## Zombie

| Code | HTTP | Title | Common Causes |
|---|---|---|---|
| `UZ-ZMB-001` | 402 | Zombie budget exceeded | Daily dollar budget hit. Raise `budget.daily_dollars` in `TRIGGER.md`. |
| `UZ-ZMB-002` | 500 | Zombie agent timeout | Agent timed out processing an event. Check the activity stream. |
| `UZ-ZMB-003` | 424 | Zombie credential missing | A required vault credential is absent. Add it with `zombiectl credential add <name>`. |
| `UZ-ZMB-004` | 500 | Zombie claim failed | Zombie could not be claimed from the database. Verify the `zombie_id` exists and status is `active`. |
| `UZ-ZMB-005` | 500 | Zombie checkpoint failed | Session checkpoint write to Postgres failed. |
| `UZ-ZMB-006` | 409 | Zombie name already exists | Name taken. Kill the existing zombie first, then re-install. |
| `UZ-ZMB-008` | 400 | Invalid zombie config | `TRIGGER.md` config_json fails schema validation. Verify trigger, tools, credentials, and budget fields. |
| `UZ-ZMB-009` | 404 | Zombie not found | No zombie with this ID in the workspace. |
| `UZ-ZMB-010` | 409 | Zombie already stopped or killed | This zombie is already stopped or killed. Re-install before issuing another stop. |
| `UZ-ZMB-011` | 400 | SKILL.md and TRIGGER.md disagree on `name:` | Top-level `name:` must match across both files. One identity per zombie bundle. |

## Credentials

These fire on the inference path — when the executor needs to call the model and can't resolve a usable LLM provider key. Not the same as workspace vault credentials your zombie uses for `http_request` (those are [`UZ-VAULT-*`](#vault) and [`UZ-ZMB-003`](#zombie)).

| Code | HTTP | Title | Common Causes |
|---|---|---|---|
| `UZ-CRED-001` | 503 | Anthropic API key missing | Workspace LLM API key not found in `vault.secrets` (key: `anthropic_api_key`). Set via the workspace credentials API, or set `ANTHROPIC_API_KEY` on the worker for dev. |
| `UZ-CRED-003` | 503 | Platform LLM key missing | No active platform-managed LLM key for the resolved provider. The platform admin must rotate the key, or your tenant must attach its own via the workspace credentials API. |

## Vault

| Code | HTTP | Title | Common Causes |
|---|---|---|---|
| `UZ-VAULT-001` | 400 | Credential data must be a non-empty JSON object | `POST /credentials` body must include a `data` field that is a JSON object with at least one key. Bare strings, arrays, scalars, and `{}` are rejected. |

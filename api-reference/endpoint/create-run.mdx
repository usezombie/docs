---
title: "Create run"
description: "Submit a spec and start a new run."
api: "POST /v1/runs"
---

## Request body

<ParamField body="spec" type="string" required>
  The spec content to execute. Must pass schema validation.
</ParamField>

<ParamField body="workspace_id" type="string" required>
  The workspace to run the spec against.
</ParamField>

## Response

<ResponseField name="run_id" type="string">
  Unique identifier for the created run.
</ResponseField>

<ResponseField name="status" type="string">
  Initial run status. One of `QUEUED`, `RUNNING`, `COMPLETED`, `FAILED`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of run creation.
</ResponseField>

<ResponseExample>
```json
{
  "run_id": "run_01J8XVKA3M",
  "status": "QUEUED",
  "created_at": "2026-03-29T14:22:00Z"
}
```
</ResponseExample>

## Deduplication

If a run with identical spec content already exists in the workspace and is still in progress, the API returns a `DUPLICATE_RUN` error instead of creating a new run. Completed or failed runs do not block new submissions.

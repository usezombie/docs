---
title: "Retry run"
description: "Retry a failed run."
api: "POST /v1/runs/{run_id}:retry"
---

## Path parameters

<ParamField path="run_id" type="string" required>
  The unique identifier of the run to retry.
</ParamField>

## Response

<ResponseField name="run_id" type="string">
  Unique identifier for the new retry run.
</ResponseField>

<ResponseField name="status" type="string">
  Initial status of the retried run. Always `QUEUED`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of the retry creation.
</ResponseField>

<ResponseExample>
```json
{
  "run_id": "run_01J8XVKB7N",
  "status": "QUEUED",
  "created_at": "2026-03-29T15:10:00Z"
}
```
</ResponseExample>

<Note>
  Retry is only available for runs in `FAILED` status. Attempting to retry a run in any other state returns an `INVALID_STATE_TRANSITION` error.
</Note>

## Why UseZombie exists

You want an AI agent to debug your homelab, triage your side projects, or audit your self-hosted stack. You do not want to hand it your kubeconfig, your GitHub token, or your SSH keys.

Giving an agent cluster-admin credentials is a risk that does not evaporate just because the model is well-behaved today. Hallucinations, prompt injections, and context-bleed into logs are real attack surfaces. A single `kubectl delete ns media` run at 3am by a well-meaning but confused agent is a very long evening.

UseZombie fixes this with three properties, together:

- **Tool policy in prose** — each zombie's `SKILL.md` is the system prompt. It tells the agent in plain English which verbs to use and which to avoid (e.g. "only `kubectl get`, `describe`, `logs`, `top`; never `delete`, `apply`, `secrets`"). No YAML allowlists, no per-skill sub-directories — the prose is the contract.
- **Placeholder credentials** — the agent process never sees your real API token. It gets a UUID placeholder, and a proxy at the network boundary swaps it for the real secret outside the sandbox. Prompt-injection attacks that try to exfiltrate the token succeed only in repeating the placeholder.
- **Customer-network workers** — the worker that actually executes tool calls runs inside your network (a Docker container in your homelab, a Pi behind your firewall, your laptop). The UseZombie control plane never routes to your cluster API.

If you're thinking "this is what Microsandbox does for code execution" — yes, similar pattern, credit where it's due. UseZombie extends it with audit + approval layers aimed at homelab, ops, and developer-tool workloads.

## How the runtime works

<Columns cols={2}>
  <Card title="Sandboxed runtime" icon="lock">
    bwrap + landlock isolation. Network deny-by-default. Only allowlisted domains reachable.
  </Card>
  <Card title="Credentials hidden" icon="shield-halved">
    Agents never see API keys. The vault injects at the boundary — the agent asks to call the tool, UseZombie makes the real request.
  </Card>
  <Card title="Customer-network workers" icon="server">
    Workers run where your infrastructure lives. Your credentials never leave your network.
  </Card>
  <Card title="Activity stream" icon="chart-line">
    Every action timestamped and queryable. `zombiectl logs` shows what happened, when, and why.
  </Card>
  <Card title="Spend ceiling" icon="gauge-high">
    Per-day and per-month dollar budgets. Tenant balance debited only on completed runs.
  </Card>
  <Card title="Kill switch" icon="power-off">
    `zombiectl kill` stops any agent mid-action. Checkpoint saved, no data lost.
  </Card>
</Columns>
